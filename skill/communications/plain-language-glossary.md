# Comms: Plain-Language Glossary

Plain-language definitions for translating Solana incident terms into words
users and non-technical stakeholders understand. Use these in public updates and
support replies; keep them accurate but jargon-free.

## Terms

```text
Transaction signature  A receipt ID for an action on Solana; safe to share.
Failed transaction     An action that didn't complete; usually no funds moved.
Wallet                 The app that holds your keys and signs actions.
Seed phrase / private key  The secret that controls a wallet. Never share it.
Signing                Approving an action; only signing can move funds.
Token approval         Permission you grant a site to use certain tokens.
Program (smart contract)  On-chain code that runs the protocol.
Upgrade authority      Who is allowed to change that on-chain code.
Mint authority         Who can create new units of a token.
Freeze authority       Who can freeze a token account.
RPC                    A service apps use to read/write the chain.
Explorer               A website to view on-chain activity.
Confirmed vs finalized Levels of certainty that an action is permanent.
Drainer                A malicious site/script that tricks you into signing away
                       funds.
```

## Usage

- Replace a jargon term with its plain definition the first time it appears in a
  public message.
- Never let a simplification become an overclaim — "usually no funds moved" is
  honest; "your funds are safe" needs evidence.
- Pair with [`public-status-update.md`](public-status-update.md) and
  [`support-macros.md`](support-macros.md).
